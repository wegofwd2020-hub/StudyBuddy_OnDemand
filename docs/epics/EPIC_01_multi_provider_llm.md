**Status:** ✅ Complete (F-1 through F-5, 19 tests, migration 0043)

---

## What it is

Allow schools to choose which LLM provider generates their curriculum content
(Anthropic Claude, OpenAI GPT-4o, Google Gemini), and optionally run two providers
in parallel to compare outputs side-by-side before approving a version.

## Original design note

The pipeline previously was Claude-only. The call was centralised in `_call_claude()` in
`pipeline/prompts.py`. The rest of the architecture was already provider-agnostic:

- Content Store schema is provider-neutral (JSON files, same shape regardless of provider)
- `content_subject_versions` already tracks multiple versions per subject — adding a `provider` column turns version comparison into provider comparison
- The version diff UI already exists in the admin console
- The review/approve workflow already gates publishing — provider comparison fits naturally into the same step
- Pipeline is async/Celery — running two providers in parallel is an orchestration change, not an architecture change

**Verdict from design doc:** Pragmatic. The existing architecture is unusually well-positioned for this. An AI agent (dynamic, runtime) was ruled out — the year-long content stability requirement requires the existing batch + version + approve model.

## Why it matters

- **Procurement unblocking:** Schools with Microsoft or Google vendor agreements may be restricted from using Anthropic. Provider choice removes that objection.
- **Data residency:** Some districts require EU/local data processing. Provider choice lets schools select a compliant option.
- **Quality comparison:** A school can run Claude and GPT-4o on the same unit and approve whichever output their teachers prefer.
- **Risk hedging:** No single-provider dependency.

## Open questions

1. **Which providers to support at launch?** All three (Anthropic, OpenAI, Gemini), or Anthropic + one other to start?
2. **Pricing pass-through:** Do schools pay for their own API keys, or does StudyBuddy absorb provider costs and charge a flat fee? Affects the billing model significantly.
3. **Quality floor:** If a provider produces content that fails the AlexJS check or JSON schema validation, do we surface that in the UI or silently retry?
4. **Default provider:** Does StudyBuddy have a preferred/default provider, or is it always school-configured?
5. **Existing content migration:** Schools with approved content generated by Claude — does adding a provider column break anything?
